Assignment 6

Completion requirements

Overview

This assignment explores some of the C++ Standard Library's containers, iterators, and algorithms. While this assignment is straight-forward:

  • you may need to look up / re-read materials in order to use and understand such better
  • the determination of the "largest" inverse frequency histogram std::set's .size(), while a bit weird, is done the way it is done in order to require you to write in a certain way to aid your learning and understanding of how to use iterators, algorithms, etc.

Should you have questions, be sure to see a TA/GA or the instructor during office hours. If sending an email, ensure you attach program code and in the email describe what your issues/questions are.

You are (strongly!) advised to NOT write the entire assignment program "in one go". Write small portions of the assignment code ensuring that they are correct before adding more code. Do make a point to understand what is actually being done, what types are being used, which functions are being called + what those functions do and return, etc. Writing small portions of the assignment code and testing such before continuing also means you will be writing "testing" code and will need to remove such before submitting your assignment.

Task

#include files

Start by #including the following:

#include <algorithm>    // e.g., std::for_each, std::sort, std::find             
#include <cstddef>      // e.g., std::size_t                                     
#include <functional>   // e.g., std::greater                                    
#include <fstream>      // e.g., std::ifstream                                   
#include <iterator>     // e.g., std::iterator_traits, std::forward_iterator     
#include <iostream>     // e.g., std::cout, std::clog                            
#include <map>          // e.g., std::map                                        
#include <set>          // e.g., std::set                                        
#include <string>       // e.g., std::string                                     
#include <vector>       // e.g., std::vector

and realize that you must only use the types and functions outlined below in your solution.

ASIDE: There are often several ways of doing the same thing. This assignment requires you to do some things in a specific way.

Writing output()

Your code must have a function called output with this prototype:

template <std::forward_iterator FwdIter>                                                      
std::ostream& output(                                                            
  std::ostream& os,                                                              
  FwdIter first,                                                                 
  FwdIter last,                                                                  
  bool const with_indices=false                                                  
);

Notice:

  • the last function parameter has a default argument value of false
  • the template parameter has a requirement imposed on it i.e.,
    • FwdIter must satisfy the std::forward_iterator concept
    • If the template parameter is a type that does not satisfy the std::forward_iterator concept then this template definition will not be defined.

ASIDE: C++20 introduced concepts and supporting syntax. This allows one to apply requirements to template parameters. Such can be done in more than one way --the above shows one.

A note about the default function parameter:

  • If you've a prototype in your code for output(), the first mention of output (i.e., the prototype) is the only mention that can have the default function parameter. This means that the definition that appears later cannot have that default function parameter. (If it does, the compiler will report an error.)
  • If you don't have a separate prototype for the output() function, then ensure that the default function parameter appears with it.

Inside the output() function:

  • Write this type alias:
    • using count_type = typename iterator_traits<FwdIter>::difference_type;
      • C++ defines in <iterator> a std::iterator_traits<T> type that allows one to obtain useful information about the iterator type T. See cppreference.com's std::iterator_traits<T> page, as well as, Section 9.5 in TCPPSL.
      • This type alias is to save typing with the next two declarations. (One could simply type in typename iterator_traits<FwdIter>::difference_type each time such is needed.)
      • The "typename" keyword is needed since the scope resolution operator, "::", is being used to access dependent identifiers through a template parameter. In general, the problem of determining of what is being referred to is a type or a variable through the scope resolution operator and template parameter(s) is an undecidable problem. To avoid such, the C++ language requires programmers to write "typename" in front of such expressions when the result is a type. (If such is not done, the compiler assumes the accessed identifier is a variable.)
  • Declare a variable of type count_type called index and explicitly default-construct it (so it has a value of zero).
  • Declare a variable of type count_type called rindex and explicitly construct it with the value returned by calling std::distance(first,last).
    • ASIDE: std::distance() is a function that computes the distance between the iterators first and last. (See TCPPSL, ATOCPP , and cppreference.com.)
  • Using the std::for_each <algorithm> function call, process the input range [first,last) with the function called being a lambda function written as follows:
    • Capture any variables defined outside the lambda function by reference.
    • Write the value passed to the lambda function to the stream os.
    • If the with_indices variable is true then write to the stream os:
      • an opening parenthesis, '(', followed by
      • the index variable followed by
      • a colon, ':', followed by
      • a negated rindex value followed by
      • a closing parenthesis, ')'
    • After the if statement write a ' ' character to the stream os.
    • Return the stream os.

The goal of output() can be seen in the Sample Program Runs section later in this assignment. Specifically, this function outputs an iterator range [first,last) to the stream os with each element output with a space character following it. If the with_indices function parameter is false, then that is all it does. If with_indices is true, then, within parentheses after the value, it outputs that entry's 0-based index followed by a ':' followed by its "reverse" 0-based index where the latter is the index obtained by counting from 0 starting the last position and traversing the range in reverse while incrementing the number.

Some tips:

  • Consider using postfix ++ with index.
  • Consider using prefix -- with rindex. (Notice that it is initiailized with the largest (positive) value it needs.)
  • Consider using unary - with the "reverse index" value to make such negative.

ASIDE: Why the C++20 concept (i.e., std::forward_iterator) use? To show that such can be usefully used. If you try to call output() passing a type for the second and third arguments that doesn't satisfy the concept std::forward_iterator<T> the code will not compile (and the compiler will output which constraints were not satisfied). Concepts will be used more in the rest of this course.

The Rest of the Program

The rest of the program can be written in main() as well as using functions that you choose to create. You must use the containers and functions calls as stated and required below in your implementation and you cannot substitute such with other equivalent code.

Handling main()'s Arguments and Reading in Data

At the top of main() should be the following:

  • main() requires exactly one argument passed to it: the path to a text file that this program will process.
    • If a path is not passed to main, then output to std::cerr: cerr << "Usage: " << argv[0] << " <text_file_path>\n"; and return 1 from main() to indicate a failure to the calling program.
  • Open the file (per the argument passed to main()) using std::ifstream.
  • Read in the file using a string variable as follows:
    • Add each word read in to a frequency histogram, i.e., std::map<std::string,std::size_t>. (Remember to increase the count each time a specific string is read in!)
    • Append each word read in to a std::vector<std::string> variable.
    • NOTE: Don't do this twice. Build the vector and map in a single pass.
    • ASIDE: By default when reading a string from a file, each string stops being read with the first whitespace character. (Recall that this is what fscanf() does as well.) Do *not* use std::getline() --just read in a string. No strings are std::quoted(), etc. --just read in std::string values. :-)
  • If the words vector is empty, then output:
    • cerr << "There is no data. Aborting.\n"; with a return 2 from main() to indicate a failure to the calling program.

At this point you are done with reading from the file. This means the words vector and the frequency histogram have everything needed. Since the words vector is not empty, there was at least one word read in and that means there is at least one entry in the frequency histogram. The latter is important since some checks that otherwise would need to be done won't have to be done.

ASIDE: For testing/debugging purposes, you can output the vector<string> using output(). It is always a good idea to be sure input has been read in properly. :-)

Reversing The Words

Now do the following:

  • Declare a std::vector<std::string>. (This will be called revwords below.)
  • Call std::reverse_copy() to process the std::vector<std::string> used to read in all words in the file earlier. The third parameter (i.e., the output iterator) needed will be to call std::back_inserter() passing in revwords.
    • When done, revwords will have all words stored in reverse order.
    • NOTE: You are not permitted to do this any other way: you must use std::reverse_copy() and std::back_inserter() with the two std::vector<std::string> objects.

ASIDE: There is certainly more than one way to copy, in reverse, everything in a container. This assignment requires you to use std::reverse_copy() with std::back_inserter() to do this. Also leave the in-order words vector alone --you will use this vector again near the end of this assignment's program. (This means you will now have a revwords vector whose values are in the reverse order of the original vector used to read in all of the words.)

Computing the Inverse Frequency Histogram

With the frequency histogram you built earlier, compute the inverse of such, i.e.,

  • Recall a frequency histogram counts the number of times something occurred, e.g., the number of types a word appears in a file. The inverse of such is therefore, a mapping of the number of times something occurred to a container of those "somethings" that occurred that many times.
  • The frequency histogram's type was std::map<std::string,std::size_t>. The inverse will also be a map. Figure out what its type needs to be.
    • NOTE: For the container you must use a std::set. Why? The code below requires sorting such --so this might as well be done implicitly by using a std::set here! :-)
  • Write the code to compute the inverse of the frequency histogram data with these requirements:
    • Declare a variable i and i_end to be the begin() and end() iterators of the frequency histogram.
    • Iterate using a traditional for loop over the range [i, i_end).
      • ASIDE: The previous assignment had you use range for loops. This loop is requiring you to use a traditional for loop with iterators instead. Although not needed here, sometimes one needs access to the iterators themselves. When that occurs, range for loops cannot be used.
    • Within the for loop you MUST follow these requirements:
      • The for loop body must be EXACTLY one statement (i.e., one line of code).
        • ASIDE: This is a requirement and is NON-NEGOTIABLE. Ask for help if you need such. Do look at what is being done and you will see why this is being required here --even though it can be done using multiple statements. (If you need to write the code using multiple statements first --do that to figure things out. Then convert such to be exactly one statement suitable for submitting this assignment.)
      • Call the .insert() member function of your inverse frequency histogram object passing in the size_t value as the key and a default-constructed set (suitable for the value).
        • The default-constructed set will be empty. Clearly a word always needs to be added to such.
        • ASIDE: When insert() returns it either added what was passed to it, or, it did not add anything. Since .insert() returns a std::pair<iterator,bool> and we always *need* to add the word to the std::set, there is no need to check the bool value if the entry was added or not since the iterator value is always correct in both cases. (Ask yourself and explore why this is the case.) This means we can use the iterator returned to access the entry to .insert() the word in to the std::set associated with this size_t value! (Cool, eh?!!)
      • After the ')' of the .insert() call determine which of first or second need to be used to access the iterator portion of the std::pair returned by .insert().
      • After the previous (sibling) bullet, detemrine how to access the value (not the key) the iterator points to.
        • ASIDE: Look up std::pair in TCPPSL especially how it is used with std::map. Clearly the iterator points to what is stored inside the container --which in this case is a std::map.
        • ASIDE: Which of . or -> must you use to access such as well? Remember the iterator "points" to something. So if you use . with the iterator you are accessing the iterator's member functions. If you are using -> and there is an overloaded -> operator for that type (and there is such for all iterators), then one is accessing what is being "pointed" to when -> is used. The same is true for, std::unique_ptr, std::shared_ptr, std::optional, etc., etc.
      • After the previous (sibling) bullet, invoke .insert() on the std::set value passing in the current frequency histogram's key.
        • ASIDE: The "current" value is determined by the current iterator value being used with this for loop. Remember you are copying data from the frequency histogram to the inverse object. The difference is that the key of the inverse is comprised of the values from the original frequency histogram; similarly the values in the inverse are comprised of the keys from the original frequency histogram.

TIP: At this point, even though you will delete this code before submitting this assignment, it is wise to check that your inverse frequency histogram is computed properly: output the original frequency histogram and the inverse to confirm that all is okay before continuing further.

Determining the Largest Number of Words Having the Same Frequency

This "largest" determination is a bit artificial. Why is it being done this way? To help ensure you do not use any "shortcuts" to aid learning and understanding of the related underlying notions. For exampe, the definition of "largest" doesn't use an already defined ordering on the std::pair's in the inverse frequency histogram map, therefore, this requires you to pass in a comparison function to std::max_element() instead of relying on a default (which would use operator<).

ASIDE: Know std::pair has no default ordering at all. Don't ever define one in your code for std::pair --keep it the way it is defined.

Before continuing, remember that a map stores its entries as std::pair values. (Re-read TCPPSL to review such.) At this point in the assignment, we won't need the size_t value anymore (which is the "key" value in the inverse frequency histogram map). The std::set of word values will be processed here. In particular, one wants to determine an entry in the inverse frequency histogram map that has a largest .size() number of words.

ASIDE: The terms "an entry" and "a largest size" are used since a largest entry is all that matters here. If there are five equal largest entries then any one of those five entries can be selected in this assignment. This makes using std::max_element() appropriate with simpler code logic since one doesn't need to be concerned with equal largest entries.

  • Determine the largest value in the inverse frequency histogram that has the largest std::set .size() value.
    • You must use std::max_element() to do this.
      • ASIDE: Since this is ordering things by the map's values --not ts keys--, one need to iterate over all elements in the map, i.e., using std::max_element() is appropriate since nothing that is being looked at is sorted.
      • ASIDE: Since you want the larger .size() values, you need to pass in a comparison function. Use a lambda function to do this. (You cannot use std::greater, etc. since this is accessing a component of the (inverse) map's entries.)
      • With the iterator to the largest position, output to std::cout:
        • "frequency ", followed by the largest position's frequency, followed by " occurs with these strings: ", followed by what calling output() on the value of what the largest position iterator points to.
          • ASIDE: The largest position iterator is what std::max_element() returns and you *know* there is at least one element so there will be a valid maximum value --so there is no need to check if the largest position is the end iterator or not.
          • ASIDE: Look at the sample run output to see what this outputs with the various input files.
Determining the "Smallest" and "Largest" Word at that Frequency

Using the largest position iterator found using std::max_element() above, "extract" / "copy" the smallest and the largest words in the std::set that the largest position iterator "points" to. Some tips on doing this:

  • Use std::set's .begin() member function and point to the resutl to get the smallest word.
  • Use std::set's .end() member function to get the one past the largest word position. Call std::prev() on it to get the previous position. Point to such to get the largest word.

Store both of these results in variables. (It is assumed these are stored in variables called "lo" and "hi" below.)

Output these variable's values to std::cout as follows:

  • output the value of lo after "lo value: " on its own line
  • output the value of hi after "hi value: " on its own line
Finding the First Position "lo" Appears in revwords + an Adjustment

Use std::find() on revwords (i.e., from the begin() to the end()) searching for the value in the variable lo. Store the value std::find returns in a variable. (This variable will be referred to as "lopos" below.)

ASIDE: lopos is the first position in revwords where the value of lo was found.

In order to not have the next call start at lopos, increase lopos by one if lopos is not equal to the end() of revwords.

ASIDE: This can be done with an if statement. It is very important with iterators to not increment them past the end() position --so one cannot do a ++lopos unless one is absolutely certain it will not be moving to one position after end().

Since it is possible for there to be multiple consecutive values of lo after lopos in revwords, this assignment has you skip to one past the last consecutive lo value (if any) by calling std::find_if_not() as follows:

  • Call std::find_if_not() on the input range [lopos, revwords.end()).
  • Pass in a lambda function that captures lo by reference and compares lo to the value passed to the lambda argument using operator==.

i.e., such will find the first position where that position's value is not equal to lo. :-)

Finally, compute the (0-based) index from revwords.begin() to lopos using std::distance. Store the result in a variable called "lo_index".

Finding the Last Occurrence of "hi" in revwords + an Adjustment

Similarly to finding the first occurrence of lo and then jumping ahead to the first non-lo value after that, the same will be done at the end of the list --a little differently.

Instead of using std::find() using rbegin(), etc. you will instead:

  • use std::find_end()
  • over the range [revwords.rbegin(),std::reverse_iterator(lopos))
  • with the second range [&hi,&hi+1)
  • and store the result in a variable called "rhipos" (for "reverse hipos").

The [&hi,&hi+1) is a clever "hack". Should one want/need an iterator range with exactly one element, one can pass in the address of a variable and one past the address of that variable as the iterator range. Since it is legal for pointers to point to one past the last element, this is okay to do. Nicely, this trick allows one to not have to put a single variable in to a container simply to use it in a single-element iterator range to be used with Standard Library algorithms. :-)

std::reverse_iterator is an adapter class that takes an iterator and uses that iterator in the reverse direction. Normally, one iterates from the begin() to the end() direction; a reverse_iterator iterates from the end() to the begin() direction offset by one when using * and -> so that rbegin() appears to be the last element, and rend() cannot be accessed.

ASIDE: The TCPPSL describes this in detail in Section 9.4.1 (Reverse Iterators). Please re-read such to fully understand what is going on.

Since rhipos will either be pointing to the last value with hi, or, to the iterator posiiton std::reverse_iterator(lopos), to have rhipos be one position after the former, write an if statement as follows:

  • if rhipos is not equal to std::reverse_iterator(lopos) then ++rhipos.

Again, this is needed because:

  • one cannot go to a position before lopos
    • ASIDE: Doing so would be very bad. The correctness of the code assumes that lopos is either before or the same as rhipos.base().
  • one wants to be after the last hi value (starting from the end() and to not be before lopos).

Finally, compute the (0-based) index from revwords.begin() to rhipos.base() using std::distance. Store the result in a variable called "hi_index".

Sorting in Descending Order

Call std::sort() on the range [lopos, rhipos.base()) with a comparison function that will sort the data in descending order.

Producing the Final Outputs

Output to std::cout "All words reversed: " and call output() to output to std::cout all words (i.e., the original vector of words read in, in order at the start of main()) from rbegin() to rend() followed by a newline.

  • Pass the value true to the fourth function parameter so the indices are produced in output().

Finally, output to std::cout "All words reversed with [" followed by lo_index followed by a comma, ',', followed by hi_index, followed by ") sorted: ", and call output() to output to std::cout all revwords, followed by a newline.

  • Don't pass the value true to the fourth function parameter so no indices will be produced in output().
