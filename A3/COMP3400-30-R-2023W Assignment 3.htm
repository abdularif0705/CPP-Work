Overview

The purpose of this assignment is to explore some object-oriented programming in C++ and a simple use of std::shared_ptr.

Prof. Q has asked you to write some C++ code to make it easier for the research group to output XML data for which many different XML structures will be needed. To start, Prof. Q has asked you to write code that will output XML documents as follows:

  • There is only one "root" document type called "notes".
  • At the top level of the root document, there can be any number of "priority" and "note" elements (i.e., many persons often call elements tags).
  • The priority element only contains a single double value.
  • The notes element has 4 child elements called "to", "from", "subject", "message" with each having some string data output.

Since this is the start of development Prof. Q has said you can assume perfect input, i.e., there is no need to worry about having the proper XML name structure and escaping. You can also assume all string data is ASCII and there will be no non-printable characters in strings. (This simplifies what you need to do since you don't need to worry about checking strings for proper structures and properly escaping XML output, i.e., you can simply appropriately store and output all strings.)

Prof. Q has also said the output is not to be formatted to be human-readable. (This means you won't have to worry about newlines and indenting the nested elements, e.g., see Sample Program Output below.)

Finally Prof. Q has said that your code must permit the creation of XML document node structures using object-oriented programming. Such must be done in a way that is easy to add more node structures (for different nodes/document types) later.

Task

Start by writing these header files which will be used in your code:

#include <cstddef>        // for std::size_t
#include <iostream>
#include <memory>
#include <ostream>
#include <string>
#include <utility>        // for std::move, std::forward
#include <vector>

(Such was provided for convenience so you can quickly focus on object-oriented programming notions in C++. That said, always look things up to learn which headers are needed in your code and to know what you are using / calling.)

Prof. Q realized that you are new at programming in C++ and has provided the definitions to start (i.e., put these in your program immediately after the header files):

class xml_node_base
{
public:
  virtual ~xml_node_base() = default;

  void output(std::ostream& os) const
  {
    do_output_open(os);
    do_output_body(os);
    do_output_close(os);
  }

protected:
  virtual void do_output_open(std::ostream& os) const = 0;
  virtual void do_output_body(std::ostream&) const { }
  virtual void do_output_close(std::ostream& os) const = 0;
};

using xml_node_base_t = std::shared_ptr<xml_node_base>;
using xml_node_bases_t = std::vector<xml_node_base_t>;

template <typename T, typename... Args>
inline xml_node_base_t make_xml_node(Args&&... args)
{
  return std::make_shared<T>(std::forward<Args>(args)...);
}

where:

  • xml_node_base is the base class all XML node types are derived from
  • Since OOP requires the use of pointers, the xml_node_base_t type alias is defined to be std::shared_ptr<xml_node_base> so that all pointers are managed (and reference counted --although that is not exploited in this assignment).
  • Since XML nodes can have children, xml_node_bases_t is defined to be a std::vector<xml_node_base_t> to make it easier to deal with children.
  • make_xml_node<T>(args...) is defined so that T is a type and args... are the arguments used to construct an instance of that type. This is done to make it easier to create instances of various XML nodes since traditional/general OOP requires instances to be pointers. (This is not exploited in this assignment --but Prof. Q requires such for future coding efforts.)

Within the xml_node_base class:

  • output() is a non-virtual const function that always calls the do_output_*() functions with the stream passed to it.
    • There is no need to make output() virtual as these functions will always be called.
    • By defining output(), this simplifies logic relating to producing output. (XML document nodes have open and close elements (tags) as well as data between such. If output() was not defined, one would have to manually call these three functions --which would be a source of programming issues/mistakes/etc. By defining output() such issues are completely avoided since the logic of output is encapsulated (hidden) appropriately with the derived classes (due to protected scope).
  • do_output_body() is not abstract since it is appropriate to have nodes with no data between the opening and closing elements (tags).
  • do_output_open() and do_output_close() are defined to be pure virtual (abstract) because each element has its own name (and potentially XML attributes that are not dealt with in this assignment) so derived classes need to define such.

The rest of the "Task" section outlines details concerning each class that must be written. (Your code can be written in the order these classes are given.)

class xml_node

An xml_node represents an named element (tag) without any data. Write the code for this class as described below:

  • Define xml_node as a class.
  • The xml_node class must inherit virtually and publicly from xml_node_base.
    • ASIDE: You want only ONE instance of xml_node_base so you need to use virtual here. To keep xml_node_base's members' scopes the same, public needs to be used since C++ classes will otherwise inherit privately by default.
  • In private scope: Declare std::string member variable to hold this node's name;
  • In public scope: The default constructor must be deleted, write: xml_node() = delete;
    • ASIDE: By writing a constructor or function prototype followed by = delete, you are telling the C++ compiler that such cannot be used, generated, etc. ever. Trying to do so will trigger a compile-time error. The default constructor is being deleted since the user of xml_node must provide a name (i.e., invoke one of the other constructors).
  • In public scope: Write a constructor, xml_node(std::string const& name) where:
    • There is nothing within the constructor body braces --you initialization MUST be before the opening brace of the constructor body!
    • In the constructor member initialization list, initialize the private std::string name variable to the std::string parameter passed to the constructor.
  • In protected scope: Override void do_output_open(std::ostream& os) const and:
    • Output via the std::ostream parameter a '<', followed by the private std::string member, followed by a ,>'.
      • e.g., "<note>" (with no quotes of course) would be output if the node's name was "note".
  • In protected scope: Override void do_output_close(std::ostream& os) const and:
    • Output via the std::ostream parameter "</" followed by the private std::string member, followed by a '>'.
      • e.g., "</note>" (with no quotes of course) would be output if the node's name was "note".

class xml_node_with_children

xml_node defined a node that has no children or data. There are times when a node will have child nodes. The xml_node_with_children defines an xml_node with children leveraging inheritance. Write the code for this class as described below:

  • Define xml_node_with_children as a class.
  • The xml_node_with_children class must inherit publicly from the xml_node class. (Do not use virtual.)
  • In private scope: Declare a member variable of type xml_node_bases_t to hold the child nodes. (This is referred to as the children data member below.)
  • In public scope: Delete the default constructor --one should never use this type wihtout passing a std::string name to its constructor.
  • In public scope: Define the constructor: xml_node_with_children(std::string const& name) by:
    • having no code in the constructor body,
    • after the constructor initialization list, colon first write xml_node{name}  (or xml_node(name)) to "call" the parent class' constructor that accepts a string, and,
      • ASIDE: This is like using super in Java to invoke a parent class' constructor --except super doesn't exist in C++. (It wouldn't make sense either since C++ supports multiple inheritance. Which class would be called when super was used?)
    • don't worry about initializing the children data member.
      • If you want you can default construct such --but if omitted, the compiler will default construct it for you. :-)
  • In public scope: Define the constructor: xml_node_with_children(std::string const& name, std::size_t reserve) by:
    • in the constructor initialization list, invoke the single-parameter std::string constructor in this class by writing xml_node_with_children{name}
       (you can use braces or parentheses with constructor syntax), and,
      • ASIDE: Again, this is like calling another constructor in the same class in Java albeit with different syntax. This is sometimes how you can avoid duplicating code when you have a number of constructors.
    • in the constructor body, invoke the private children data member's .reserve() function passing in the reserve value to it.
      • ASIDE: The reserve(n) member function in std::vector and some other types allocates RAM sufficient for n entries --but it doesn't create elements. This allows the vector to not have to be resized (and copied/moved internally to do this) unless more then n elements are added to it.
  • In public scope: Define the constructor: xml_node_with_children(std::string const& name, xml_node_bases_t children) by:
    • in the constructor initialization list, invoke the parent class' constructor passing the std::string parameter to it,
    • in the constructor initialization list, construct the children private data member by moving the xml_node_bases_t parameter passed to the constructor, and,
      • i.e., write children_{std::move(children)} if your private data member is called "children_" and your xml_node_bases_t parameter is called "children".
      • The xml_node_bases_t parameter was passed by value --which makes a copy of the original. For efficiency, you want to move the contents of that copy in to the private data member. If you didn't move it, it would be copied and the first copy would be destroyed. That would be a waste/inefficient.
        • ASIDE: FYI, the caller of this constructor could have moved data in to this parameter so the "copy" made isn't always an extra copy. If the argument had its data moved into it, then the "copy" isn't really a copy --otherwise it is.
    • the constructor body must be empty.
  • In protected scope, define these two overloads:
    • auto& children() { return children_; };
      auto const& children() const { return children_; };
    • NOTE 1: You cannot call your private data member in this class "children" since this member function uses that name. Rename your private data member if you did call it "children".
    • NOTE 2: children_ in these definitions are what I called my private data member. Substitute the name you used.
    • NOTE 3: Notice that both of the overloads return references --but the const overload returns a const& since the underlying object is const.
  • In protected scope, override the function void do_output_body(std::ostream& os) const by writing the following code in it:
    • write a range-for loop to iterate through all children (i.e., the private data member variable) with each element's type being auto const& (you want the const& part; auto is just convenience to avoid writing xml_node_bases_t), and within the range-for body, using -> with the element reference variable to call output(os).
      • ASIDE: Recall that std::shared_ptr and other smart pointer types overload * and -> so the object can be used like a pointer. Essentially ->output(os) is invoking xml_node_base's output() function.

class value_node<T>

In many instances one might want an xml_node with a simple datum. value_node<T> defines itself in terms of an xml_node but adds the ability to output its stored instance of type T in the body. Write the code for this class as described below:

  • Define value_node<T> as a class by writing:
    • template <typename T>
      class value_node
  • The this class must inherit publicly from the xml_node class. (Do not use virtual.)
  • In private scope: Define a data member of type T to hold the datum (which is originally passed to the constructor).
  • In protected scope: Override void do_output_body(std::ostream& os) const and output the private data member datum to the os stream.
  • In public scope: Define the constructor value_node(std::string const& name, T const& v) where:
    • in the constructor initialization list: pass the std::string name parameter to the parent class constructor,
    • in the constructor initialization list: pass the T parameter to datum (use constructor syntax to do this!), and,
      • ASIDE: Don't use std::move() here --T was passed as an lvalue. This will always make a copy. (It is possible to have other constructor overloads that accept rvalue references in order to move when things should be moved so one can have both copying and moving as is appropriate --but such isn't being done here.)
    • ensure the constructor body is empty.

class note

The node class is defined in terms of xml_node_with_children except the number and order of children is fixed. This class ensures that invariant and nicely makes it very easy to create Prof. Q's "note" elements!

ASIDE: Unlike HTML, XML does not fix the names of its elements (tags) or underlying structure. This class fixes such to something hard-coded. This is okay in a program: one wants to make it easier to use things --and this class makes it easier to create notes (albeit this assignment only outputs stuff --but you get the idea).

Write the code for this class as described below:

  • Define note as a class.
  • The note class must inherit publicly from xml_node_with_children.
  • Everything below is defined in public scope.
  • Define a deleted default constructor.
  • Define a constructor note(std::string const& to, std::string const& from, std::string const& subject, std::string const& message) as follows:
    • in the constructor initialization list, call the parent class' constructor passing these parameters: "note" and 4,
    • in the constructor body, call xml_node_with_children's children() function (you can use this-> or omit such, e.g., writing children() is sufficient) and before the semicolon write:.push_back(make_xml_node<value_node<std::string>>("to",to))
      • i.e.,append to the vector a newly constructed xml_base_node_t (which is of derived type value_node<std::string>)
    • in the constructor body, do the same as the previous line but instead of "to",to use "from",from
    • in the constructor body, do the same as the previous line but instead of "from",from use "subject",subject
    • in the constructor body, do the same as the previous line but instead of "subject",subject use "message",message

class root and std::ostream Operator

As with containers, etc. the actual "document" one wants to hold everything in is special. One should not be permitted to add the document inside of itself (at least in this assignment!). To achieve this while exploiting inheritance in order to make it easy to define things, this class does NOT use public inheritance, instead it uses protected inheritance.

ASIDE: Protected inheritance demotes everything that is public in the parent to protected scope. Everything at protected scope remains protected. By not inheriting publicly, a programmer cannot mistakenly pass instance of the root class to something that accepts xml_node_base*, etc. This is because protected inheritance is not a public "is a" relationship for things outside of the root class.

Write the code for the root class as described below:

  • Define root as a class.
  • The root class must inherit using protected --not public-- access from xml_node_with_children.
  • In public scope, write: using xml_node_with_children::xml_node_with_children;
    • This will inherit all of the constructors from the parent class as constructors of this class.
  • In public scope, write: using xml_node_with_children::output;
    • This will promote the parent class' output() function (which was defined in xml_node_base) to PUBLIC scope. (The protected inheritance demoted such to protected scope and this line promotes it to public since it is in the public section.)
  • In public scope, write: using xml_node_with_children::children;
    • This will promote the parent class' children members to public scope. (These were never declared at public scope and now the root class is making them publicly available in this way.)

Write an IOStream operator overload to output the root class as follows:

  • Remember to use std::ostream& and the proper bit shift operator to do this.
  • Remember the root type must be passed in as const& (you're outputting so treat what you are outputting as read-only!).
  • The function body must call r.output() passing in the std::ostream object to such, and, return the std::ostream object.

Some Comments

The above code is not hard but for newcomers to C++, it is perhaps annoying since one will be learning the syntax. and this will inevitably lead to compiler errors. It does help to compile your code frequently (e.g., after every few edits) ensuring there are no issues before adding more code. If you compile the entire program after you've written it, it will take a long while to fix all of the issues --so please don't write everything before compiling it! And remember if you need help, please do ask for help! :-)

If were doing a more elaborate version of this, you would have had more code to properly escape XML data, ensure all names don't have illegal characters in them (e.g., spaces, control characters, etc.) throw an exception if an illegal name was passed, etc. Such is beyong the scope of one assignment --but it is not hard to see how things can be done.
