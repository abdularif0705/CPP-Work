Overview

In Assignment 4 you wrote code to complete the provided code to read and write playing cards. In this assignment you will use what you did in Assignment 4 + some of Assignment 4's provided code + some provided code in Assignment 5 to write Assignment 5. Assignment 4's objective was to be able to have playing card types that permitted one to be able to read, write, etc. such. Now, with such, one can do more. :-)

Suppose you had boxes of decks of differently branded playing cards. (Each brand of playing card has a different back --so one does not want to play a game of cards using cards from different brands.) Unfortunately, you don't know how many decks of cards you have, and, you don't know how many brands' decks you have. Worse, somehow you accidentally dropped all of them! You and others picked up the resulting mess of cards and threw them in to small containers. In this assignment each container is a file and all such files are in one directory. That is the input to this program. (A program is provided that generates random input for this assignment.)

The objectives of this assignment are to:

  • use what you learnt in Assignment 1 to read a directory of files' information (each file contains some number of playing_cards),
  • use what you built in Assignment 4 to read/write/etc. playing_cards,
  • output the total number of cards that were "dropped" into the containers
  • output the total number of brands of playing cards
  • for each brand's decks:
    • output the total number of cards with that company brand
    • output the minimum number of complete decks of cards
    • output whether or not each deck is missing cards, and, if cards are missing output which cards are missing
This assignment will require using/writing:
  • an operator <=> overload that is not defaulted
  • a struct/class capable of storing a playing_card_company (provided with this assignment) and playing_card pair
  • a function that can read the previous bullet's struct/class in from a file
  • a function to generate a full deck of cards as a std::set<playing_card> object
  • using <filesystem> to process all files in the input directory
  • many range for loops
  • a std::map<playing_card_company,std::vector<playing_card>, i.e., a dictionary of playing card companies' playing cards (i.e., the ones read in)
  • a std::vector<std::set<playing_cards>> container, i.e., a container that holds a number of (sorted) decks
  • std::set_difference() to determine which cards are missing from a deck

Writing an operator<=> Overload for Two playing_card_company Objects

The author of class playing_card_company did not write an operator<=>() overload for playing_card_company --and you would like / need to sort them! Fortunately, the operator<=>() overload does not have to be written inside a class. (This will work because playing_card_company has public functions that can be called to get the values needed to be ordered.)

Write such as follows:

  • The parameters of the overload must both be of type playing_card_company const&.
  • The overload cannot be "= default"ed. (You must write the code --which is 1 line of code to write.)
  • The overload must return the type std::strong_ordering.
  • The single line of code uses the <=> operator with the .name() returned by the two arguments of the function.
    • NOTE: It is the case that you want the ordering to be based on something inside another object. In this case you want the ordering of playing_card_company to be done on what the company's name() is. That type (which is std::string) already has an operator<=>() overload defined --so use <=> with .name() from each parameter and return the result. No if statements, etc. are needed. :-)

Writing struct/class pcc

If you look at the sample input data (e.g., the input files and the a5-gen-input-provided.cpp file) you will notice each playing card is output followed by its brand (i.e., playing_card_company). So you'll want to write a simple class to read such in. Since the only need for this class is to read the data in AND each data member in the class is used as a value object (i.e., no pointers, references, and the objects themselves are not pointer or reference type objects), the easiest way to write this class is to declare it as a struct (which will make the data members all public by default).

To write pcc:

  • Declare a (public) member variable of type playing_card_company.
  • Declare a (public) member variable of type playing_card.

Writing std::optional<pcc> read_pcc(std::istream&)

In Assignment 4 the playing_card, card_suit, and card_face types all had "read_*" functions returning std::optional. Since all of those types cannot be default constructed (recall the default constructors were "= delete"d), you will need to write a read_pcc function that does the same.

To write read_pcc, in the function:

  • Declare an std::optional<playing_card> variable to represent a playing card.
    • Assign this variable the result of calling read_playing_card. (This code was in Assignment 4.)
  • Declare a playing_card_company variable.
    • If you've not already, look at a5-provided.hpp to see how playing_card_company is defined. Notice it can be default constructed and has a .name() function.
  • If the playing_card variable has a value (remember it is of type std::optional<playing_card>) AND you are able to read in a playing_card_company such that the stream is good. then return pcc{ card_comp_variable, *card_variable }; where the variables are your variable above.
    • NOTE 1: Since the order in the input file is playing_card followed by playing_card_company you must read them in this order.
    • NOTE 2: Since things can fail, you need to test such. Recall you can if (is >> some_var) since the operator>>() overload returns the stream and a stream implicitly converted to a bool will return true if and only if no errors or EOF has occurred. Only if there has been no errors/EOF on input, can one be sure the data was read in properly, thus, only if such is true can you return a pcc object. (As hopefully noticed in Assignment 4, the std::optional type has an overloaded operator=() allowing it to be assigned to a value that is stores (in which case it will store a copy of that value in the object).)
  • Return std::nullopt; otherwise.

A number of you had issues in Assignment 4 with std::optional. If those continue do re-read what std::optional is for. In C++, the indirection operators, i.e., * and ->, are often overloaded to refer to something inside an object, e.g., like std::optional, or referred to by the object, e.g., iterators. Clearly, if you pass the card_variable you declared as is, you are passing a std::optional<playing_card> type as the second (aggregate member data) element of the struct/class pcc type. But that won't work since the second element is of type playing_card. You need to make these match.

ASIDE: If you declared the member variables in struct/class pcc in the other order, then you need to swap the ordering of the values passed to construct the pcc aggregate in the return statement.

With this done, you should now be able to read one card (with a company) from any of the input files. (Notice in a5-provided.hpp that playing_card_company has IOStream operator overloads. Notice those operator overloads use std::quoted for both input and output. Consider what such enables one to do easily.)

Writing std::set<playing_card> generate_full_deck()

You will need one more function. After reading in the playing cards, in order to determine which cards are missing from a deck, you will need to have a function that generates all cards in a deck --in sorted order. Why? In order to use std::set_difference(), it is required that the input be sorted. (Look up std::set_difference() in TCPPSL or cppreference.com.) While you could read such in to a std::vector or some object, you would still need to sort it, so why not generate the deck sorted in a std::deck() object in one go? :-)

To write generate_full_deck():

  • Declare std::set<playing_card> retval. This is what you need to return and you need to add elements one at a time --so declare it. :-)
  • Use a range for loop to iterate through all faces that have suits (i.e., exclude the jokers) from this iteration.
    • Within such use a range for loop to iterate through all suits.
      • Within such call .insert({ face_var, suit_var }); on the set variable to insert such.
  • After those for loops, write a range-for loop for the two jokers and insert such, or, write two statements that do the same.
    • ASIDE: Look at the playing_card class. Notice the contructors that you can call. If you use the wrong one, an exception will be thrown --which is bad.
  • Return the set variable.

Some of you might ponder how to use a range for loop to output a bunch of enumeration values. Here are some ideas:

  • You could create a container of those enumeration values which can then be used with the range for loop.
  • You could use the C++11 { ... } syntax which creates a std::initializer_list object
    • Notice that std::initializer_list objects have .begin() and .end() functions which means you can write the { ... } stuff right in the range for loop where the container would normally be placed.
    • ASIDE: This is the easiest way to do this.

Try such out!

ASIDE: I do recommend when using range for loops to normally capture elements by & or const&. If you don't write & with its type then it will be captured by value (i.e., a copy is made). For enumerations, they are just integers so it is not a big deal. With classes, etc. they likely won't fit in to a CPU register and will incur overheads with unnecesary copies made of them. Instead capture those by reference.

Reading in All of the Playing Cards

Before you can determine how many cards there are, how many card brands there are, how many full decks of each brand there are, etc. you need to read all of the cards in first.

While you could read all of the cards in a set, vector, etc. a little thought should be done first. Since you want to group all of the cards by brand, it is wiser to read all of the cards in to a dictionary type first, i.e.,

std::map< playing_card_company, std::vector<playing_card> >

i.e., the key is playing_card_company (which is what is sorted) and the value is vector<playing_card>, i.e., the playing cards belonging to that brand.

Why do this in this way? Because one does not need the playing card associated with the playing_card_company when processing playing cards within a company. (Not having such associated with the value will make things easier to do.)

ASIDE 1: While not part of this assignment, if you were to use something else, you would discover issues with playing_card's default constructor not being defined so you'd need to do things differently, etc. The code would likely be more complicated.

ASIDE 2: The design of no default constructor is actually a good thing. If you look playing_card, it is not possible to create an invalid playing card in terms of constructing a playing card. This means the playing_card class doesn't need an "is_valid()" member function that requires testing.

ASIDE 3: Recall other OOP languages store and use references to their objects. This is what allows you to refer to an object with no default constructor equivalent. In C++, objects are values by default,. If you add a * then it is a point, a & or && it is a reference, etc. Since you are using objects by value here, you cannot default construct something that cannot be. That is, in part, why std::optional is used for the read_*() functions. That said, one might not want to use std::optional too much. This assignment avoids such by having you use a std::map. (And such helps one learn how to use std::map.)

Declare a variable of the dictionary type above. (This variable will be referred to as "all_cards" below.)

With all_cards declared, you can now use a range for loop to iterate through all files in the path passed to main via argv[1].

NOTE: You must use the <filesystem> header related types/functions to do this. Look at Assignment 1 for ideas. You cannot use std::recursive_directory_iterator though. Look at cppreference.com for another available iterator.

WIthin the range for loop (assuming that loop's iteration variable is called "entry"):

  • Declare a std::ifstream variable passing in entry.path().native().
    • This will open that entry file for input.
    • You can assume in this assignment that there are only files in that directory, etc. You do not need to test if the entry is a file, etc.
  • WIth the file open, use a loop to read in using read_pcc all of the pcc entries in the file.
    • Within this loop, only add what is read in to the map, e.g., assuming what is read in is called "datum":
      • all_cards[datum->comp].push_back(datum->card)

ASIDE: Why does that line work to add things to the map? Read about map in TCPPSL. (Recall the map's value is a vector --which is why .push_back() is used.)

Outputting the Total Number of Cards

So you read in all of the cards in to the map. If you call the map's .size() function, you would get the number of key-value pairs, i.e., the number of playing card brands --not the number of cards. To obtain the number of cards, you need to add up the number of cards in every vector in the map. This is straight-forward to do:

  • You'll need an unsigned integer. Use std::size_t since there are all kinds of unsigned integer types and std::size_t is used with container sizes, etc. Initialize this integer to 0 (or explicitly default construct it with {}).
  • Use a range for loop to iterate through all_cards.
    • Within such, += the unsigned integer with entry.second.size() where entry is the for loop iteration variable.
  • Output the total number of cards.

How does this work? Range for loop iteration variables represent what is in the container. Since maps store std::pair objects with .first being the key and .second being the value, entry.second.size() accesses the value and then the .size() member of the vector<playing_cards> is invoked.

Outputting the Number of Companies and Their Names

Per the above, you know the number of companies, output such.

Now using a range for loop over all_cards, output each company name (i.e., the keys in the map) one a single line as follows:

  • Before the company name is output, output two spaces.
  • When outputting the company name ensure that std::quoted() is used to wrap such in quotation marks and escape any quote characters.
Determining the Number of Decks and Missing Cards

Finally for each brand (i.e., playing_card_company), you want to determine the number of full decks of cards you have and which cards are missing (if any). This means looping through all_cards processing each value in each iteration. (The map's values contain all of the cards read in with the branding of the map's key.) To write such:

  • Write a range for loop to iterate over all_cards. (Below it is assumed the iteration variable is called entry.) Within such:
    • Output a newline.
    • Output a line with the current playing_card_company using std::quoted() followed by " card stats:\n".
    • Output "  Total number of cards: " followed by the total number of cards (for this brand bring processed), followed by a newline.
    • Declare a vector<set<playing_card>>. (It is assumed this variable is called decks below.)
      • This will represent the decks of playing cards
      • It is initially empty.
      • As you need to add more decks, you will append such to the vector.
      • Each deck of cards might as well be sorted (i.e., that is why std::set is used) to enable std::set_difference to be used later with ease.
    • Write a range for loop to iterate over all cards in this entry's value (not key). (Below it is assumed this iteration variable is called card.)
      • Write a range for loop to iterate over all decks.
        • Within such, if the card is already in the deck, then add a new deck to decks (i.e., you will need to use .push_back({})) and then insert card to such (i.e., use .back().insert(card)). If the card was not already in the deck, insert it.
          • HINT: While you can use .find() to determine if card is in the deck, you can also use .insert() to achieve the same. Recall .insert() for std::map returns a std::pair<iterator,bool> so if .second is false then the card was not inserted in to the map.
      • Use decks to output the total number of decks needed.
      • Initialize a std::size_t variable to 1 called count. (You will used this to count the decks so you can say a deck number such as in, "Deck 1 is not missing any cards.")
      • Write a range for loop to iterate over decks. (It is assumed below that this iteration variable is called deck.)
        • Call generate_full_deck() and store the result in a variable.
        • Declare a std::vector<plahing_card> variable called missing.
        • Call std::set_difference() with the first range being begin() to end() of the full deck variable, the second range being the begin() to end() of deck, and the third argument being std::back_inserter(missing).
        • If there are no cards in missing, output "  Deck ", count, " is not missing any cards.\n".
        • Otherwise output, "  Deck ", count, " is missing: ", followed nby each card in missing separated by a space, and after all cards output a newline character.
        • Don't forget to increment count at the end of the loop.
Closing Words

While one could have used various algorithms to process the above ranges, the range for loop is easily used to do such when one needs to iterate (sequentially) over all elements in a container (or any type with begin() and end() functions returning iterators).

The above also shows how one can effectively use containers to do processing, and, one can make use of algorithms such as std::set_difference() to aid the computation of a desired answer.

Each subtask needed in this assignment was not hard to do. So while an overall set of tasks can seem daunting, often focusing on doing each subtask on its own makes things easier to do.

Nowhere in this code is there any explicit dynamic memory allocation and deallocation calls. Nowhere in this code are open files not closed, etc. (recall with RAII that the destructors clean up / close any managed resources acquired by the object). If you see C++ code that is not like this (and such is not completely inside the private parts of classes), then such is likely not well-written C++ code or the code originates from before the C++ 2011 standard.
